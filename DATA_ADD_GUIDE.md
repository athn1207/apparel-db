# データを増やす手順

アパレルDBに商品データを追加するときの流れです。

---

## 全体の流れ（3ステップ）

1. **data/data.json を編集する**（商品を追加・修正）
2. **画像を使う場合は images/ に画像を置く**
3. **Git でコミット＆プッシュ** → GitHub Pages に反映

---

## ステップ1: data/data.json を編集する

### 1-1. ファイルを開く

プロジェクト内の **`data/data.json`** を Cursor で開きます。

### 1-2. 商品の形（1件の例）

1件の商品は、次の形で書きます。

```json
{
  "id": "ブランド名-連番",
  "brand": "ブランド名（大文字など統一するとよい）",
  "product_name": "商品名",
  "price": 価格（数値のみ）,
  "images": ["images/ファイル1.jpg", "images/ファイル2.jpg"],
  "screenshot_count": 数値,
  "created_at": "YYYY-MM-DD"
}
```

| 項目 | 説明 | 例 |
|------|------|-----|
| **id** | 商品ごとにひとつだけのID（半角英数字・ハイフン推奨） | `"nike-002"` |
| **brand** | ブランド名。一覧でまとまるので表記をそろえる | `"NIKE"` |
| **product_name** | 商品名 | `"ドライフィットTシャツ"` |
| **price** | 価格（円）。数値のみ | `5500` |
| **images** | 画像のパスを配列で。なしなら `[]` | `["images/nike-002-1.jpg"]` |
| **screenshot_count** | スクリーンショット数。一覧の並び順に使う | `3` |
| **created_at** | 登録日（任意） | `"2026-02-21"` |

### 1-3. 追加のしかた

- **既存の配列 [ ] の中**に、新しい商品オブジェクトを **カンマ** で区切って追加します。
- 最後の商品の後ろに **カンマ** を打ってから、新しい `{ ... }` を書きます。

**追加前（最後の1件）：**
```json
  {
    "id": "nike-001",
    "brand": "NIKE",
    ...
  }
]
```

**追加後（2件目を足した例）：**
```json
  {
    "id": "nike-001",
    "brand": "NIKE",
    ...
  },
  {
    "id": "nike-002",
    "brand": "NIKE",
    "product_name": "ドライフィットTシャツ",
    "price": 5500,
    "images": [],
    "screenshot_count": 3,
    "created_at": "2026-02-21"
  }
]
```

- **id は重複させない**ようにしてください（同じIDが2つあると、商品ページでどれが開くか分かりにくくなります）。
- JSON の **カンマ忘れ・余分なカンマ** に注意（編集後、ブラウザでサイトを開いて表示が崩れていないか確認すると安全です）。

---

## ステップ2: 画像を使う場合

商品ページに写真を出したいときは、**2つの作業**をします。

1. **画像ファイルをプロジェクトの「images フォルダ」に入れる**（サイトが画像を読み込めるようにする）
2. **data.json のその商品の "images" に、その画像の場所を書く**（どの画像を表示するかサイトに教える）

---

### 2-1. 画像ファイルを置く

**「images フォルダ」** は、プロジェクトの中にある **`images`** という名前のフォルダです。

- エクスプローラーで **apparel-db** を開くと、**data** や **scripts** と同じ階層に **images** があります。
- 表示したい画像（.jpg / .png など）を、**その images フォルダの中にコピーまたは保存**します。
- ファイル名は **半角英数字** にすると安全です（例: `nike-002-1.jpg`）。日本語やスペースは避けた方が無難です。

**例:** 商品「nike-002」の写真が2枚ある場合  
→ `nike-002-1.jpg` と `nike-002-2.jpg` を **images フォルダの中** に置く。

---

### 2-2. data.json の "images" にパスを書く

**「パス」** は「その画像が、サイトのどこにあるか」を表す文字列です。このサイトでは **`images/ファイル名`** という形で書きます。

- **data/data.json** を開き、画像を追加したい商品の **"images"** を探します。
- もともと `"images": []` になっている部分を、次のように **画像のパスを並べた配列** に書き換えます。

**1枚だけ表示する場合:**
```json
"images": ["images/nike-002-1.jpg"]
```

**複数枚表示する場合（カンマで区切る）:**
```json
"images": ["images/nike-002-1.jpg", "images/nike-002-2.jpg"]
```

- 配列の中は **ダブルクォート " " で囲む**・**カンマで区切る** のを忘れないようにします。
- 商品ページでは、ここに書いた順に画像が **縦に並んで** 表示されます。

**画像を使わない商品** は、`"images": []` のまま（空の配列）でOKです。

## ステップ3: GitHub に反映する（サイトを更新する）

データや画像を追加・変更したあと、次のコマンドで GitHub に送ると、GitHub Pages のサイトが更新されます。

### 3-1. ターミナルを開く

Cursor で **ターミナル** を開きます（メニュー: ターミナル → 新しいターミナル）。

### 3-2. プロジェクトのフォルダに移動

```powershell
cd "c:\Users\akiko\OneDrive\デスクトップ\apparel-db"
```

### 3-3. 変更を追加してコミット

```powershell
git add .
git commit -m "商品データを追加（NIKE Tシャツなど）"
```

- **`git commit -m "..."`** の **"..."** の部分は、何を追加したか分かる短いメッセージにすると後で便利です。
  - 例: `"商品データを5件追加"` / `"AURALEEの商品を2件追加"`

### 3-4. プッシュする

```powershell
git push
```

- 初回だけ認証が出ることがあります。その後はそのまま完了するはずです。

### 3-5. サイトで確認

- 1〜2分ほど待ってから、**https://athn1207.github.io/apparel-db/** を開きます。
- 新しいブランドや商品が一覧に出ているか、商品ページで画像が表示されているかを確認します。

---

## 手順のまとめ（毎回やること）

| 順番 | やること |
|------|----------|
| 1 | **data/data.json** を開き、新しい商品を配列に追加する（id は重複させない） |
| 2 | 画像を使う場合は **images/** に画像を置き、その商品の **images** にパスを書く |
| 3 | ターミナルで **git add .** → **git commit -m "メッセージ"** → **git push** |
| 4 | 数分後に公開URLで表示を確認する |

---

## 注意すること

- **id** は全商品でひとつずつ別の値にすること。
- **brand** は表記をそろえると、一覧で同じブランドがまとまります（例: すべて `"NIKE"`）。
- **data.json** を編集したあとは、カンマや `[ ]` `{ }` の対応が正しいか確認すると安心です（サイトが真っ白になったり、一覧が出ないときは JSON の書き損じを疑ってください）。

データを増やしたら、この手順の「ステップ3」まで毎回行うと、サイトにどんどん反映されていきます。
